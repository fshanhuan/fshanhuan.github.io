git的快速学习

前期简单指令

  • git init命令把这个目录变成Git可以管理的仓库

  • git add告诉Git，把文件添加到仓库

  • git commit命令，-m后面输入的是本次提交的说明（需要使用双引号来说明注释内容）

  • git status命令看看结果

  • git diff顾名思义就是查看difference

  • git log命令显示从最近到最远的提交日志

版本控制

  • HEAD指向的版本就是当前版本，因此，Git允许我们在版本的历史之间穿梭，使用命令git reset --hard commit_id。

  • 穿梭前，用git log可以查看提交历史，以便确定要回退到哪个版本。

  • 要重返未来，用git reflog查看命令历史，以便确定要回到未来的哪个版本。

    命令git reset HEAD <file>可以把暂存区的修改撤销掉（unstage）

    git checkout -- 文件名丢弃工作区的修改，实际上是用版本库里的版本替换工作区的版本

    该种情况同时可以防止误删文件的操作

git rm用于删除一个文件

git具有特色的分支管理，可以有不同的分支来进行代码的操作。

这样就是经典的master分支，同样也可以

建立dev分支，假如我们在dev上的工作完成了，就可以把dev合并到master上。Git怎么合并呢？最简单的方法，就是直接把master指向dev的当前提交，就完成了合并

也相当于

用git branch命令查看当前分支，再git checkout master回主分支，最后删除git branch -d dev
